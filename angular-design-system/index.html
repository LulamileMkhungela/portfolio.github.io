  button {
    display:inline-flex; align-items:center; gap:var(--space-2);
    font-weight:var(--font-weight-medium);
    border:none; cursor:pointer;
    border-radius:var(--radius-md);
    transition:opacity .15s, transform .1s;
    font-family:var(--font-display);
  }

    background:var(--color-primary-900); color:var(--bg);

  }
  .ds-btn--secondary {
    background:transparent; color:var(--ink);
    border:1.5px solid var(--border-strong);
    padding:var(--space-2) var(--space-4);
  }

    padding:var(--space-2) var(--space-4);
  }
  .ds-btn--ghost {

export class DsButtonComponent {
  @Input() variant: 'primary'|'secondary'|'destructive'|'ghost' = 'primary';
  @Input() size: 'sm'|'md'|'lg' = 'md';
  @Input() disabled = false;
  @Input() loading = false;
  get classes() {
    return `ds-btn ds-btn--${this.variant} ds-btn--${this.size}`;
  }
}

End-to-End Process

How a component
goes from Figma
to shipped code.

Five stages. One rule per stage. Nothing moves forward until the current stage is signed off. Names never change between stages — that's the entire system.

01
Figma Variables
Design source of truth
Design every component with Auto-layout and Figma Variables. Name all component properties to exactly match the planned Angular @Input names — this is the contract. Every spacing and colour annotation uses a token name (space/4, not 16px). Gate: design lead review and sign-off on all states.
Auto-layoutComponent propertiesToken annotations4+ states minimumWCAG contrast check
02
Token Studio
JSON export
Export Figma Variables to structured JSON using the Tokens Studio plugin. The JSON key path mirrors the Figma variable path exactly — color/primary/900 becomes color.primary.900 in JSON. Gate: PR review on tokens/tokens.json. Zero manual edits allowed.
tokens.jsonPR reviewNo manual editsGit-tracked
03
Style Dictionary
CSS + TypeScript transform
Style Dictionary runs in CI on every token commit. One transform rule: dots become hyphens. Output: variables.css (CSS custom properties) and tokens.ts (TypeScript constants). Gate: automated test confirms all expected variables are present with correct values. No manual CSS writing.
CI pipelinevariables.csstokens.tsAutomated testsZero manual CSS
04
Angular / Codebase
Standalone component build
Angular 17 standalone component consuming the generated CSS variables. Selector format: ds-{name}. Every @Input name matches the exact Figma component property name. SCSS uses only token variables — no hardcoded values ever. Gate: code review + axe-core accessibility check in CI + semantic versioning.
Angular 17 standaloneARIA embeddedaxe-core CISemvernpm publish
05
Storybook
Living documentation
Storybook story path matches Figma component path exactly. Every variant has a dedicated story. Controls knobs mirror @Input names. Compodoc auto-generates the prop table from TypeScript. A component is only marked stable once all five stages are green in this tracker. Gate: design sign-off comparing Storybook output vs Figma spec.
Storybook 7CompodocDesign sign-offPath = Figma pathDeploy on merge
